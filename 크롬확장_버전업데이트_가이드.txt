========================================
크롬 웹스토어 확장프로그램 버전 업데이트 가이드
========================================

■ 현재 버전: 1.0.1 (manifest.json 기준)

----------------------------------------
1. 버전 번호 올리기
----------------------------------------
manifest.json에서 version 값을 수정합니다.

예시:
  "1.0.1" → "1.0.2"  (작은 수정/버그 수정)
  "1.0.1" → "1.1.0"  (기능 추가)
  "1.0.1" → "2.0.0"  (큰 변경)

규칙: 반드시 이전 버전보다 커야 합니다 (예: 1.0.1 다음은 1.0.2 또는 1.1.0 등).


----------------------------------------
2. 패키지(zip) 만들기
----------------------------------------
업로드할 때 필요한 파일만 포함한 zip을 만듭니다.

★ 반드시 포함:
  - manifest.json
  - background.js
  - content.js
  - injected.js
  - admin.html
  - admin.js
  - firebase-config.js
  - icons/ (icon16.png, icon48.png, icon128.png)

★ 제외 (zip에 넣지 말 것):
  - README.md, *.txt 가이드 등 (선택)
  - .git, node_modules 등
  - create_icons.html, 소스만 있는 파일

만드는 방법:
  1) naver-blog-converter 폴더 안에서
  2) 위 목록 파일/폴더만 선택
  3) 우클릭 → "압축" / "ZIP으로 압축"
  또는
  - 폴더 전체를 zip으로 압축한 뒤, 불필요한 파일만 zip 안에서 삭제


----------------------------------------
3. Chrome 웹스토어 개발자 대시보드 접속
----------------------------------------
1) Chrome 웹스토어 개발자 콘솔
   https://chrome.google.com/webstore/devconsole

2) Google 계정으로 로그인

3) "네이버 블로그 HTML 변환기" 항목 클릭
   (이미 등록한 항목이 있어야 함. 처음이면 "새 항목"으로 먼저 등록)


----------------------------------------
4. 새 버전 업로드
----------------------------------------
1) 해당 확장프로그램 페이지에서
   "패키지" 또는 "새 버전 업로드" 버튼 클릭

2) "업로드 새 패키지" 선택

3) 2번에서 만든 zip 파일 선택 후 업로드

4) 업로드 후 표시되는 내용 확인
   - 버전 번호가 올라간 것인지
   - 경고/오류가 없는지


----------------------------------------
5. 변경 사항 요약 작성
----------------------------------------
"패키지 업로드" 후 또는 "상품 정보"에서
"변경 사항" / "What's new" 란에 사용자에게 보일 업데이트 내용을 입력합니다.

예시:
  - 1.0.2: 사용량 표시 개선, 버그 수정
  - 1.0.1: 최초 공개

한국어와 영어 모두 적어두면 좋습니다.


----------------------------------------
6. 검토 요청 및 배포
----------------------------------------
1) 모든 필수 항목 입력 후 "제출하여 검토" (Submit for review) 클릭

2) 검토는 보통 며칠 소요될 수 있습니다.

3) 승인되면 사용자에게 자동으로 업데이트가 배포됩니다.
   (사용자는 Chrome가 확장 프로그램을 자동 업데이트합니다.)


----------------------------------------
요약 체크리스트
----------------------------------------
□ manifest.json version 값 올림 (예: 1.0.2)
□ 필요한 파일만 포함한 zip 생성
□ 개발자 콘솔에서 "업로드 새 패키지"로 zip 업로드
□ 변경 사항 요약 입력
□ "제출하여 검토" 클릭
